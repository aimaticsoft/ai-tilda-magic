

# План экспорта сайта Aimatic в Tilda (блоки T123)

## Обзор задачи

Требуется экспортировать весь сайт, разработанный на React + Framer Motion + Tailwind, в формат, совместимый с блоками **T123 (HTML-блок)** в Tilda. Каждый блок сайта станет отдельным HTML-фрагментом со встроенными стилями и скриптами.

---

## Структура блоков для экспорта

Сайт содержит **12 отдельных блоков**, которые нужно конвертировать:

| № | Блок | Файл-источник | Особенности |
|---|------|---------------|-------------|
| 1 | Header (навигация) | Header.tsx | Фиксированный, мобильное меню |
| 2 | Hero (главный экран) | HeroSection.tsx | Particles canvas, parallax |
| 3 | Marquee (бегущая строка) | ParallaxText.tsx | Бесконечная прокрутка |
| 4 | About (о компании) | AboutSection.tsx | Timeline с анимациями |
| 5 | Services (услуги) | ServicesSection.tsx | Карточки 3x2 |
| 6 | Cases (кейсы) | CasesSection.tsx | Бесконечная карусель |
| 7 | Demo (ИИ-агенты) | DemoSection.tsx + AgentAvatar | Карусель с кастомными аватарами |
| 8 | Advantages (преимущества) | AdvantagesSection.tsx | Две секции карточек |
| 9 | Reviews (отзывы) | ReviewsSection.tsx | Карточки с рейтингом |
| 10 | FAQ (вопросы) | FAQSection.tsx | Аккордеон |
| 11 | Contacts (контакты) | ContactsSection.tsx | Форма + контакты |
| 12 | Footer (подвал) | Footer.tsx | Ссылки, копирайт |

## Архитектура решения

Создам **генератор экспорта**, который:
1. Конвертирует Tailwind-классы в inline CSS
2. Преобразует Framer Motion анимации в CSS @keyframes
3. Сохраняет JavaScript для интерактивных элементов
4. Генерирует готовые HTML-фрагменты для каждого блока

## Техническая реализация

### Этап 1: Базовые стили (CSS Variables)
Создам единый файл стилей, который нужно вставить в **первый блок** или в настройки Tilda:

- CSS переменные цветов (--primary, --accent, --background и т.д.)
- Базовые классы (.glass-card, .btn-neon, .heading-primary)
- Keyframes анимаций (fadeUp, pulseGlow, float)
- Шрифт Inter (Google Fonts)

### Этап 2: Конвертация компонентов
Для каждого блока создам:

1. **Статический HTML** — разметка без React
2. **Scoped CSS** — стили с уникальными префиксами (`.aimatic-hero-*`)
3. **Vanilla JS** — для интерактивности:
   - Карусели → Swiper.js или чистый JS
   - Particles → Canvas скрипт
   - Scroll-анимации → Intersection Observer
   - Magnetic buttons → mousemove events

### Этап 3: Страница экспорта
Новая страница `/export` с интерфейсом:
- Список всех блоков с чекбоксами
- Превью выбранного блока
- Текстовое поле с готовым HTML-кодом
- Кнопка «Копировать код»
- Кнопка «Скачать все блоки»

## Особенности конвертации анимаций

| Framer Motion | Vanilla эквивалент |
|---------------|-------------------|
| `animate={{ y: [0, -15, 0] }}` | CSS `@keyframes float` |
| `whileHover={{ scale: 1.05 }}` | CSS `:hover { transform: scale(1.05) }` |
| `useInView` | Intersection Observer API |
| `useScroll` + `useTransform` | `scroll` event + CSS variables |
| Бесконечная карусель | CSS `@keyframes marquee` или JS `setInterval` |

## Инструкция для пользователя (после реализации)

1. Откройте страницу `/export` в превью
2. Скопируйте код **«Базовые стили»** и вставьте в Tilda:
   - Настройки сайта → Дополнительный CSS (или первый T123 блок)
3. Для каждой секции:
   - Добавьте блок **T123** в Tilda
   - Скопируйте код нужного блока
   - Вставьте в T123
4. Проверьте мобильную версию и при необходимости подкорректируйте

## Ограничения и альтернативы

**Что будет работать идентично:**
- Все визуальные стили и цвета
- Hover-эффекты и переходы
- Базовые scroll-анимации
- Карусели (с использованием JS)
- Мобильная адаптивность

**Что может отличаться незначительно:**
- Плавность spring-анимаций (CSS transitions vs Framer Motion springs)
- Magnetic button эффект (упрощённая версия)
- Particles background (упрощённая версия для производительности)
