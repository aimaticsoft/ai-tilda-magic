

## Проблема

Две причины, почему сайт не работает как ожидается:

### 1. Данные аналитики не сохраняются (ошибка RLS)

Каждые 30 секунд трекер пытается сохранить данные в базу через `upsert`, но получает ошибку 401. Причина: `upsert` с `ON CONFLICT` требует разрешение на **SELECT** (чтобы проверить, есть ли уже запись с таким `session_id`). Сейчас SELECT разрешён только для `service_role`, но не для `anon`.

Решение: добавить RLS-политику SELECT для `anon` на таблицу `site_visits`.

### 2. Публикация сайта

Бэкенд-изменения (миграции, edge functions) применяются автоматически. Но фронтенд (код, стили, компоненты) обновляется только после нажатия "Update" в диалоге публикации. Нужно переопубликовать проект.

### План действий

| Шаг | Что делаем |
|-----|-----------|
| DB миграция | `CREATE POLICY "Allow anonymous reads" ON public.site_visits FOR SELECT TO anon USING (true);` |
| Публикация | После применения миграции — нажать "Update" в диалоге публикации |
