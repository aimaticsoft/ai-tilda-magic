
Цель: убрать из Telegram все технические подписи типа `button`, `button.pir`, `a[...]` и показывать клики только понятными русскими названиями.

1) Почему сейчас так происходит
- В трекере есть fallback, который при пустом тексте клика возвращает `tag.class` (например, `button.pir`).
- В ряде мест на сайте есть иконки-кнопки без текстовой подписи (стрелки каруселей, точки, мобильное меню), поэтому и попадают технические метки.

2) Что сделаем (для вас)
- В уведомлениях Telegram клики будут читаться по-русски и структурно: что нажали, где нажали, сколько раз.
- Технические слова (`button`, `a[...]`, классы) полностью исключим из вывода.
- Даже если фронтенд где-то отдаст “сырой” клик, бэкенд его нормализует в русский формат.

3) Технический план
- `src/hooks/useUserTracker.ts`
  - Переписать извлечение названия клика с приоритетом:
    1) `data-analytics-label` / `aria-label` / `title`
    2) видимый текст (`innerText`, с нормализацией пробелов)
    3) для ссылок — человекочитаемое русское имя по домену/пути
    4) финальный fallback: `Кнопка без подписи` + контекст секции (например, `в блоке «Кейсы»`)
  - Полностью убрать fallback `tag.class` (чтобы `button.pir` больше не появлялся).
- `src/components/Header.tsx`, `src/components/CasesSection.tsx`, `src/components/DemoSection.tsx`
  - Добавить русские `aria-label` на иконки-кнопки (prev/next, точки, меню и т.д.).
- `src/components/ProductsSection.tsx` (и при необходимости другие карточки)
  - Добавить `data-analytics-label` на кликабельные карточки/ссылки для коротких понятных названий.

  - Добавить server-side нормализацию кликов:

  - Сгруппировать вывод по шаблону: `• Название действия — N раз` + отдельный блок “неопознанные клики” (если останутся).
