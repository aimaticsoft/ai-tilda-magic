## Анализ сайта Aimatic — предложения по улучшению

Я изучил весь сайт: Hero, About, Products, Services, TargetAudience, Cases, Calculator, Demo, Advantages, Reviews, FAQ, Contacts, Footer, Header, SEO-разметку, Edge Functions и конфигурацию. Вот комплексный план улучшений, разбитый по категориям.

---

### 1. Конверсия и доверие

**A. Секция «Процесс работы с нами» (How We Work)**  
Добавить визуальный таймлайн из 4 шагов: Заявка → Аудит → Внедрение → Результат. Снимает страх «а что будет после оплаты?» и показывает прозрачность. Каждый шаг — с иконкой и коротким описанием

**B. Логотипы клиентов / партнёров**  
Добавить горизонтальную полосу с логотипами компаний-клиентов (или отраслей) в зону между Hero и About. Классический элемент social proof.

---

### 2. Интерактив и вовлечение

**C. Кнопка «Наверх» (Back to Top)**  
Сайт длинный — добавить плавающую кнопку для возврата к началу, появляющуюся после скролла.

---

### 3. Безопасность и стабильность

**D. CSP-заголовки (Content Security Policy)**  
Настроить заголовки безопасности для защиты от XSS и инъекций через Edge Function, обслуживающий заголовки.

---

### 4. Производительность и SEO

**E. Предзагрузка критических ресурсов**  
Добавить `<link rel="preload">` для шрифтов и логотипа в `index.html`, чтобы ускорить первый рендер.

**F. Web Vitals мониторинг**  
Добавить отправку метрик Core Web Vitals (LCP, FID, CLS) в Яндекс.Метрику для отслеживания реальной производительности.

**G. Мета-теги для продуктовых страниц**  
Обновить Schema.org разметку: добавить `SoftwareApplication` для каждого продукта (AimSales, AimVisual, AimSeller, AimAcademy).

---

### 5. UI/UX улучшения

**H. «Липкая» CTA-кнопка на мобильных**  
На мобильных устройствах добавить фиксированную кнопку «Написать нам» внизу экрана, которая всегда видна. Это значительно повышает конверсию на мобильных.

**I. Микроанимация успеха на форме**  
После отправки формы — анимация «галочка + конфетти» вместо простого toast. Создаёт приятное впечатление.
